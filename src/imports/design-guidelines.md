Design Guidelines for Figma
Design this app as a calm, credible, health-aware consumer product focused on helping users understand and reduce nighttime artificial light exposure.

The app should not feel clinical, alarmist, or overly medical. It should feel:

calm

intelligent

trustworthy

modern

slightly futuristic

behavior-change-oriented

The core value is simple: Make the invisible light environment visible, then help the user act on it.

1. Product Intent
The design should support a product that helps users:

notice when artificial light exposure is high

reduce additional screen-based light burden

understand patterns over time

take simple, immediate action

The experience should prioritize awareness + action, not passive tracking.

2. Core Features to Design Around
Feature 1: Estimated Artificial Light Exposure
Purpose
Help users become aware of their cumulative artificial light exposure in a simple, understandable way.

Design note
Do not frame this as a precise medical measurement. Present it as an estimate, score, load, or trend.

Suggested UI labels

Light Load

Night Light Exposure

Exposure Score

Artificial Light Estimate

Avoid labels such as

retinal damage tracker

eye damage meter

absorbed radiation score

Feature 2: High Artificial Light Alerts
Purpose
Alert users when they enter environments with high artificial light exposure, especially at night.

Design note
Alerts should feel helpful, not fear-based. Focus on clarity and next-step actions.

Every alert should answer

what is happening

why it matters

what the user can do now

Feature 3: Auto Screen Adjustment
Purpose
Reduce extra screen strain and nighttime blue-rich light disruption by adapting screen brightness and tone.

Design note
This is the most action-oriented feature and should feel seamless, smart, and reversible.

The UI should always show

what changed

why it changed

how to override it

Feature 4: Weekly Recap With Tips
Purpose
Show trends, habits, and personalized tips so users can improve over time.

Design note
This should feel like a reflective wellness summary, not a spreadsheet.

3. Design Principles
A. Calm Over Panic
The app should never look like a hazard warning system unless there is a genuinely high-risk context. Use soft emphasis and measured language.

B. Action Over Raw Data
Every important data point should connect to an action.

Examples

high exposure → reduce brightness

bright commute → use Night Mode earlier

repeated evening spikes → change bedtime phone habits

C. Simplicity Over Scientific Clutter
Avoid overly technical charts, medical diagrams, or dense terminology in the main UX. Advanced details can live behind secondary screens.

D. Personal Over Generic
Recommendations and summaries should feel specific to the user’s behavior, timing, and environment.

E. Transparent Over Magical
When the app makes adjustments or raises alerts, explain why. Users should not feel that the app is behaving randomly.

4. Brand and Tone Direction
The visual language should sit between:

a wellness app

a modern utility app

an environmental intelligence tool

The tone should feel:

reassuring

smart

non-judgmental

lightly premium

grounded in real-world use

Avoid

hospital aesthetics

biohazard or warning-heavy visuals

cyberpunk overload

generic productivity dashboard styling

5. Visual Style
Color Direction
Use a palette that reflects light conditions and circadian rhythm.

Suggested structure

Deep navy / midnight tones for backgrounds and nighttime context

Soft amber / warm gold for healthy or reduced-light states

Cool white / pale blue accents only for exposure or bright-light indicators

Muted coral or orange for alerts that require attention

Avoid highly saturated neon colors unless they are used sparingly for emphasis.

Lighting Metaphor
The UI should subtly communicate the idea of moving from harsh light to balanced light. Use gradients, glow, blur, softness, and atmospheric layering carefully.

Surface Style
rounded cards

soft shadows

layered panels

minimal borders

comfortable spacing

low visual noise

6. Typography
The typography should be clean and contemporary.

Recommended style

highly legible sans serif

large, calm headlines

compact but readable labels

a strong hierarchy between metric, explanation, and action

Use typography to separate:

current status

explanation

recommendation

Metrics should feel important, but never aggressive.

7. Iconography and Illustration
Use icons sparingly and clearly.

Useful icon themes

sun / moon transitions

screen brightness

location pin

alert pulse

trend graphs

sleep / bedtime cues

smart home controls

Illustration style should be minimal and ambient. Avoid cartoonish health mascots.

8. Core Screens to Design
1. Onboarding
Goal
Explain the problem simply and build trust.

Must communicate

artificial light exposure can affect comfort, habits, and nighttime routines

the app helps track patterns and reduce unnecessary exposure

some features use estimation, location, and device behavior

Onboarding should include permission flows for

location

notifications

display adjustment, if supported

Trust cues to include

estimate-based language

a brief explanation of how data is used

user control over automation

2. Home / Dashboard
Goal
Give the user a quick read on current status and the next best action.

Must include

current exposure state

current environment status

whether screen adjustment is active

one clear recommendation

Suggested card structure

Current Light Load

Environment Status

Screen Protection Status

Today’s Pattern

3. Alert Screen
Goal
Explain environmental risk and provide immediate actions.

Must include

alert level

what triggered it

why it matters

what the user can do next

Possible actions

dim screen

warm display

mute future alerts for a period

mark this place as routine

view tips

Tips should appear here in concise form.

4. Weekly Recap Screen
Goal
Turn data into understanding and habits.

Must include

total weekly exposure trend

highest exposure times

highest exposure locations or contexts

comparison to the previous week

2–3 personalized tips

This screen should feel reflective and rewarding. Use trend summaries, not dense analytics.

5. Settings / Controls
Goal
Make automation feel user-controlled.

Must include

alert sensitivity

auto-adjust on/off

preferred sleep hours

tip frequency

privacy controls

connected devices / smart home later

9. Information Architecture
Recommended primary navigation

Home

Alerts / Activity

Recap

Settings

Alternative, if a more insight-led structure is preferred

Today

Alerts

Trends

Profile

Keep navigation small and obvious. Do not overload the app with too many tabs.

10. UX Guidance by Feature
For Exposure Tracking
Show trends and ranges, not false precision.

Prefer

Low / Moderate / High

Exposure Score: 72

Above your usual evening range

Avoid giving the impression of medical diagnosis.

For Alerts
Use a clear three-part pattern:

status

reason

action

Example structure

High nighttime light detected

You’re in a bright, blue-heavy environment

Reduce screen intensity or limit exposure for the next 20 minutes

For Screen Adjustment
Always include:

an indicator that adjustment is active

a one-tap override

a brief “why this changed” explanation

For Tips
Tips should be:

short

contextual

behavior-based

practical

Good tip examples

Most of your exposure spikes happen after 10 PM.

Your commute appears brighter than your home environment.

Your screen stayed bright during multiple nighttime alerts.

11. Data Visualization Guidance
Charts should be elegant and easy to scan.

Use

line charts for trends

bar charts for time-of-day exposure

heatmaps for exposure by hour and day

simple scorecards for summaries

Avoid

technical medical charts

overly crowded dashboards

visualizations that require explanation to decode

Key rule
Every chart should answer one obvious question.

12. Accessibility
This is especially important because the app deals with light and visual comfort.

Design for

strong text contrast

large tap targets

readable chart labels

dark-mode-first thinking

reduced animation where needed

no reliance on color alone for meaning

Because the app is about visual strain, the UI itself should never become visually exhausting.

13. Motion and Interaction
Motion should feel subtle and atmospheric.

Good motion ideas

smooth brightness-state transitions

ambient glow changes

soft card elevation changes

gentle graph animations

Avoid

flashing alerts

jittery motion

aggressive pulsing

anything that mimics visual stress

14. Trust and Credibility Guidelines
The design should reinforce scientific humility.

Use language such as

estimated exposure

likely bright environment

based on your recent patterns

recommended adjustment

Avoid language such as

your eyes absorbed dangerous radiation

damage detected

retina risk confirmed

eyesight loss predicted

This is critical. The product should feel credible, not exaggerated.

15. Differentiation and Moat to Reflect in Design
The design should communicate that this app is more than a brightness toggle.

The moat should feel like:

personalized environmental intelligence

adaptive intervention

trend-based coaching

context-aware alerts

The app should look like it understands:

where the user is

when exposure matters most

what behavior can help immediately

16. MVP Design Priority
If only a few screens are fully designed, prioritize them in this order:

Home / Dashboard

Alert Screen

Screen Adjustment State

Weekly Recap

Onboarding

Settings

17. Deliverables for Figma Exploration
The first design pass should include:

moodboard

color palette

type scale

icon direction

one onboarding flow

one home screen

one alert screen

one weekly recap screen

one settings screen

a component library for cards, alerts, charts, and toggles

18. Final Creative Direction Summary
Design the product like a calm light-awareness companion for urban life.

It should feel like it quietly notices when modern light environments become excessive, helps the user adapt in real time, and gives them simple ways to improve over time.

The app should not feel like a medical scanner or a doom machine. It should feel like a smart layer between the user and an overstimulating world.
